🎯 本课目标：理解 E2Map 如何融入完整的导航系统（目标选择 → 路径规划 → 控制执行 → 事件处理 → 重规划），以及实验三大场景的设计和结果。

1. 完整导航系统流程

论文给出了一个清晰的算法流程（Algorithm 1）：

Algorithm 1: Navigation with E2Map in the real world

# ① 目标选择：语言指令 → 目标列表
goal_list = GoalSelector(l_inst)

# ② 初始化定位
p_agent = Localization(P_0)

# ③ 对每个目标依次执行
for goal in goal_list:
    # ④ 用 E2Map 情绪值作为代价规划路径
    tau = Planning(p_agent, goal, M)  # D* 算法
    while not arrived(goal):
        a_t = MPPI(p_agent, tau)  # 控制
        agent.run(a_t)
        p_agent = Localization(P_t)
        # ⑤ 事件检测与处理
        if event occurs:
            l_evt = F_ed(I_{t-h}, I_t, I_{t+h}, p_ed)  # 事件描述
            s_emo = F_ee(l_evt, p_ee)  # 情绪评估
            p_update = get_update_pose(p_agent)  # 获取位置
            M.update(s_emo, p_update)  # 更新 E2Map
            tau = Planning(p_agent, goal, M)  # 重新规划
        t = t + 1

1.1 目标选择器（Goal Selector）

输入：用户语言指令 \(l_{inst}\)（如"去冰箱拿瓶水，然后放到桌上"）

工作流程：

  1. Llama3 LLM 将指令转换为代码，调用目标选择 API
  2. API 通过 CLIP 文本编码器计算物体名称与 VLMap 中视觉特征的相似度
  3. 聚类去噪后，选取相似度最高的物体集群作为目标
  4. 结合指令中的空间信息（如"旁边"），选择目标周围的具体网格位置

1.2 路径规划与控制

模块算法作用
路径规划D* 算法 [36]以 E2Map 情绪值为代价，生成最小代价路径。E2Map 更新后，D* 可以增量式重规划，无需从头计算。
运动控制MPPI [37]模型预测路径积分控制，计算精确控制量使机器人沿路径行驶。
定位3D LiDAR [38]基于三维激光点云的实时位姿估计。
💡 与势场法的区别：E2Map 用情绪作为代价看起来很像是势场法中的"斥力"，但论文特意指出：为了规避势场法固有的局部极小值问题，E2Map 使用独立的规划器（D*）而不是将目标作为势场吸引力。这是一个重要的工程决策。

2. 实验设置

2.1 环境

  • 仿真环境：ROS Gazebo，使用 3D 扫描仪扫描真实会议室并建模仿真
  • 真实环境：首尔国立大学会议室，布置沙发、椅子、桌子、冰箱、微波炉
  • 机器人：拥有 LiDAR、RGB 摄像头、深度相机的移动机器人

2.2 三大场景

⚠️ 危险标志 (danger sign)

类型：环境变化

初始建图后引入新的静态障碍物（危险标志）。测试对环境变化的适应能力。

🚶 人-墙 (human-wall)

类型：动态事件

人形物体从墙后突然走出。测试对动态事件的行为调整能力。

🚪 动态门 (dynamic door)

类型：动态事件

机器人接近时门意外打开。测试对突发事件的响应（仿真仅限、真实环境因安全原因排除）。

2.3 基线方法

  • LM-Nav [23]：基于拓扑图的导航，使用三大基础模型
  • VLMaps [9]：使用视觉语言地图进行开放词汇导航

3. 实验结果与分析

3.1 仿真环境结果

方法静态环境⚠️ 危险标志🚶 人-墙🚪 动态门
LM-Nav [23]5/101/100/101/10
VLMaps [9]10/100/100/100/10
E2Map（本文方法）10/109/109/109/10
📊 关键发现：
  • 在静态环境中，E2Map 和 VLMaps 成绩相同（10/10）——说明 E2Map 不牺牲静态性能
  • 在三个动态场景中，E2Map 大幅领先——说明自反思机制确实有效
  • 基线方法"反复撞同一个障碍物"——没有学习能力
  • LM-Nav 偶尔成功是因为其拓扑图目标选择有时恰好选到了不需要经过障碍物的目标位置——这是一种运气，不是能力

3.2 真实世界结果

方法静态环境⚠️ 危险标志🚶 人-墙
E2Map（本文方法）10/109/109/10

真实世界结果与仿真高度一致，证明了方法的可迁移性和实用性。

3.3 行为调整的可视化

论文通过 Fig. 6 展示了关键的可视化结果：

  • 事件前：E2Map 中墙的情绪影响范围较小
  • 事件（碰撞）后：墙的情绪影响空间范围显著扩大
  • 行为变化：机器人经过墙边时保持安全距离，避免再次与人碰撞

4. 局限性分析

论文坦诚了若干局限性，这在组会中是很值得讨论的点：

局限性说明可能的改进方向
依赖传感器/仿真器触发事件目前需要外部信号告知"事件发生了"，而非自主检测集成异常检测算法
主要评估了负面情绪虽展示了正面情绪示例，但实验未涉及扩展到正面情绪任务
动态门场景只在仿真中测试真实环境中担心损坏设备和机器人更安全的碰撞避免机制
每次更新需要 LLM/LMM 调用可能引入延迟和成本轻量化模型或边缘部署
