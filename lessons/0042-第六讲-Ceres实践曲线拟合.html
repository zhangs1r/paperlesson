本课目标

你不是只要知道“Ceres 可以跑通”，而是要明白：残差在哪里写、参数块在哪里交给求解器、自动求导替你省掉了什么。

和前文怎么接

上一节的 L-M 方程还停留在推导层面，这一节我们把它交给一个成熟优化库。你会第一次感受到“最小二乘目标函数”在代码中是什么样子。

原书选的练手问题是：

$$ y = \exp(ax^2 + bx + c) + w, $$

其中 $a,b,c$ 是待估参数，$w$ 是高斯噪声。于是问题就变成：

$$ \min_{a,b,c} \frac{1}{2}\sum_{i=1}^{N} \left\| y_i - \exp(ax_i^2 + bx_i + c) \right\|^2. \tag{6.28} $$

这和上一节的通用形式 $\min \frac{1}{2}\|f(\boldsymbol{x})\|^2$ 已经完全对上了。只是现在：

  • 优化变量是三个参数 $a,b,c$；
  • 每个数据点给出一个一维残差；
  • 总目标函数就是所有残差的平方和。

注意书里特别提醒了一句：待估的是 $a,b,c$，不是 $x$。这句话看起来简单，但其实是在训练你区分“观测自变量”和“真正的优化变量”。

1. Ceres 的工作方式：你给残差，它替你做迭代

书里把 Ceres 定位成一个“面向通用最小二乘问题的求解库”。它最一般的形式是：

$$ \min_x \frac{1}{2}\sum_i \rho_i\left(\|f_i(x_{i_1},\dots,x_{i_n})\|^2\right), \qquad l_j \le x_j \le u_j. \tag{6.27} $$

对于这一节的曲线拟合，我们其实用的是最朴素版本：

  • 没有上下界约束；
  • 没有鲁棒核函数 $\rho$；
  • 就是普通的误差平方和。

所以你可以先把 Ceres 看成三层分工：

你负责

写清楚“一个观测对应的残差长什么样”。

Ceres 负责

把所有残差块拼成目标函数，并根据选项选择 L-M 或其他策略迭代优化。

自动求导负责

在你不手写雅可比的前提下，帮你拿到求解器需要的一阶信息。

这一节最值得体会的，不是 Ceres 某个类名，而是这种分工方式：目标函数定义和求解器控制被很好地拆开了。这也是它和 g2o 在风格上的第一层差异。

2. 代码结构逐段讲：这不是“抄代码”，而是在学最小二乘接口

2.1 第一步：定义一个残差 Functor

书里的核心代码是这个仿函数：

struct CURVE_FITTING_COST {
  CURVE_FITTING_COST(double x, double y) : _x(x), _y(y) {}

  template <typename T>
  bool operator()(const T* const abc, T* residual) const {
    residual[0] = T(_y) - ceres::exp(
      abc[0] * T(_x) * T(_x) + abc[1] * T(_x) + abc[2]
    );
    return true;
  }

  const double _x, _y;
};

如果把这段代码翻成人话，就是：

它在做什么

给定一组当前参数估计 $abc=[a,b,c]$，以及某个样本点 $(x_i,y_i)$，计算这个样本点的残差：真实观测减去模型预测。

这里最值得注意的是模板参数 T。书里说 Ceres 会像调用函数一样去用这个对象，而模板化的写法使它既能用来正常求值，也能服务于自动求导。

2.2 第二步：把每个观测加成一个残差块

ceres::Problem problem;
for (int i = 0; i < N; i++) {
  problem.AddResidualBlock(
    new ceres::AutoDiffCostFunction<CURVE_FITTING_COST, 1, 3>(
      new CURVE_FITTING_COST(x_data[i], y_data[i])
    ),
    nullptr,
    abc
  );
}

这里可以把它理解成：每个样本点都向总目标函数贡献一个一维误差块。模板参数 <1,3> 的意思正是：

  • 残差维度是 1；
  • 参数块维度是 3。

这和书里前面讲的“参数块”概念也正好衔接上了。虽然这一节只有一个参数块，但到了第十讲 BA，这种“把局部误差项挂到参数块上”的组织方式会变得非常关键。

2.3 第三步：配置求解器

ceres::Solver::Options options;
options.linear_solver_type = ceres::DENSE_QR;
options.minimizer_progress_to_stdout = true;

ceres::Solver::Summary summary;
ceres::Solve(options, &problem, &summary);

书里这里没有深挖每一个选项，而是更想让你建立一个正确直觉：Ceres 的大量行为都是通过 Options 配置出来的。也就是说，你不需要自己写 G-N 或 L-M 的迭代循环，而是把目标函数和优化选项交给它。

这也是它比很多手写优化代码更适合快速实验的原因。

2.4 自动求导为什么值钱

书里很坦诚地说了一个细节：它把 Ceres 自动求导称作“自动求导”，但又提醒你本书仍会大量介绍雅可比推导，因为那更利于理解问题，也更少出错。

对你现在来说，可以先抓住这两个层面：

工程上

自动求导让你更快把问题跑起来，尤其适合原型验证和复杂残差实验。

理论上

如果你不知道雅可比大致长什么样，出了问题时就很难判断是模型写错、尺度错了，还是求解器设置不合适。

所以这一节最好的学习姿势不是“以后都不推导了”，而是：“先让库帮我跑起来，同时不放弃理解导数结构”。

3. 原书结果图怎么读：不是只看‘拟合上了没’

这张图最值得看的不是“绿线和红线差不多”，而是三层关系：

  • 蓝色散点说明观测本身有噪声，不会乖乖落在真模型上；
  • 红线代表生成数据时的真实曲线；
  • 绿线代表通过最小二乘优化恢复出来的模型。

书里给的数值结果大致是：

$$ a \approx 0.891943,\qquad b \approx 2.17039,\qquad c \approx 0.944142. $$

和真值 $(1,2,1)$ 已经很接近，而且初始代价从大约 $1.824887\times 10^4$ 降到了大约 $5.096854\times 10^1$。这不是说明“Ceres 很神奇”，而是说明上一节讲的非线性最小二乘框架真的开始工作了。

4. 本课小结

练习本身很小

只是三个参数的曲线拟合，但它把“最小二乘问题如何交给库”讲得很清楚。

Ceres 的气质

你主要定义残差和参数块，求导和迭代控制大部分交给库配置完成。

和后文的联系

第十讲再用 Ceres 做 BA 时，问题会大很多，但工作流仍然是“定义残差块 + 配置求解器”。

课后练习

  1. 为什么这一节里每个观测点都对应一个残差块，而不是把所有点一次性写进一个大函数？
    展开查看答案
    因为最小二乘天然就是由许多局部误差项组成的。按观测逐块组织更符合求解器的目标函数结构，也更容易扩展到后面的大规模稀疏问题。
  2. 模板参数 <1,3> 在这一节里具体在表达什么？
    展开查看答案
    表示每个残差块输出一维残差，而对应的优化变量块维度是 3，也就是参数 $a,b,c$ 三个量。
  3. 这一节和第十讲 Ceres 求 BA 之间最值得提前建立的联系是什么？
    展开查看答案
    都是“定义局部残差项并交给 Ceres 统一求解”的工作流。区别只是 BA 的残差块会同时连接位姿、路标甚至相机内参，规模和稀疏性都更复杂。
