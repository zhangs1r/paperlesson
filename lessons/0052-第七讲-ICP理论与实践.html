ICP 理论与实践

到这一步，我们来到第七讲信息量最充足的一条路线：两帧都能给出 3D 点。于是问题不再是“怎么从像素撑起深度”，而是“如何把两团三维点刚体对齐”。这就是 ICP 在本书里的角色。🔄

本课目标

理解带匹配关系的 3D-3D 位姿估计为何比 2D-2D、3D-2D 更“信息充足”，并掌握书里给出的两种求法：SVD 闭式解与李代数优化解。

先把一个误区拨正

本书这里说的 ICP，是“匹配点已知”的 3D-3D 位姿估计，而不是激光 SLAM 里那种需要反复找最近邻配对的完整版 ICP。这里的问题比很多人印象里简单得多。

设两组配对好的三维点为 $\{p_i\}$ 和 $\{p_i'\}$，我们要找刚体变换 $(R,t)$ 使得：

$$ p_i = Rp_i' + t. $$

和 PnP 最大的区别在于：这里完全不需要再经过相机投影模型。换句话说，ICP 讨论的是三维空间里的刚体对齐问题，它既可以出现在视觉 RGB-D 场景，也可以出现在激光点云场景。

1. SVD 解法为什么这么漂亮

书里先给出了标准的最小二乘目标：

$$ \min_{R,t} \frac{1}{2} \sum_{i=1}^{n} \|p_i - (Rp_i' + t)\|_2^2. \tag{7.49} $$

最巧妙的一步，是引入两组点的质心：

$$ p = \frac{1}{n}\sum_{i=1}^{n} p_i,\qquad p' = \frac{1}{n}\sum_{i=1}^{n} p_i'. \tag{7.50} $$

然后把每个点改写成“质心 + 去质心偏移”。这样一拆，目标函数就会自然分裂成两部分：一部分只和旋转 $R$ 有关，另一部分在 $R$ 已知后能直接给出平移：

$$ t^* = p - Rp'. \tag{7.53} $$

于是整个 ICP 变成一条非常清楚的三步走：

算质心 → 去质心 → 先解旋转，再回代解平移

1.1 真正需要 SVD 的地方在哪

定义去质心点：

$$ q_i = p_i - p,\qquad q_i' = p_i' - p'. $$

构造矩阵：

$$ W = \sum_{i=1}^{n} q_i q_i'^T. \tag{7.56} $$

对 $W$ 做奇异值分解：

$$ W = U\Sigma V^T. \tag{7.57} $$

若 $W$ 满秩，则最优旋转为：

$$ R = UV^T. \tag{7.58} $$

这就是书里 SVD 解法的精华。它的魅力在于：不需要迭代，不需要初值，不需要担心陷入局部极小，直接把解析解端出来。

2. 代码里最容易看漏的三个点

Point3f p1, p2;
for (int i = 0; i < N; i++) {
    p1 += pts1[i];
    p2 += pts2[i];
}
p1 /= N; p2 /= N;

Eigen::Matrix3d W = Eigen::Matrix3d::Zero();
for (int i = 0; i < N; i++) {
    W += Eigen::Vector3d(q1[i].x, q1[i].y, q1[i].z) *
         Eigen::Vector3d(q2[i].x, q2[i].y, q2[i].z).transpose();
}

Eigen::JacobiSVD<Eigen::Matrix3d> svd(W, Eigen::ComputeFullU | Eigen::ComputeFullV);
Eigen::Matrix3d R_ = svd.matrixU() * svd.matrixV().transpose();
Eigen::Vector3d t_ = Eigen::Vector3d(p1.x, p1.y, p1.z) - R_ * Eigen::Vector3d(p2.x, p2.y, p2.z);

这段代码最容易被粗略扫过去，但有三个细节很值得你盯住：

先去质心

因为这样才能把平移从误差里干净地分离出去。

W 的定义不要写反

点集顺序和你约定的变换方向一致，否则最后得到的是反方向位姿。

输出时要特别留意坐标变换方向

书里也专门提醒：这里的 3D-3D 变换方向和前面 PnP 的写法相反，很容易搞混。

这一点和第三讲、第四讲一直强调的坐标系意识完全一致。很多“算法错了”的锅，其实不是算法错，而是你把 $T_{12}$ 和 $T_{21}$ 弄反了。

3. 非线性优化版 ICP：为什么结果几乎和 SVD 一样

书里继续给出了李代数形式的 ICP 目标：

$$ \min_{\xi} \frac{1}{2} \sum_{i=1}^{n} \|p_i - \exp(\xi^\wedge)p_i'\|_2^2. \tag{7.59} $$

单个误差对位姿的导数是：

$$ \frac{\partial e}{\partial \delta \xi} = -\left(\exp(\xi^\wedge)p_i'\right)^\odot. \tag{7.60} $$

这套写法和上一课 PnP 的 BA 风格非常统一：同样是李代数位姿、同样是 g2o 边、同样是 LM 迭代。差别只是观测边从 2D 重投影误差，换成了 3D 坐标误差。

但书里实验结果也告诉你一个很重要的工程结论：对于“匹配关系已知”的 ICP，SVD 通常已经给出解析最优解，所以再做非线性优化，结果往往几乎不变。

这不是说优化没意义，而是说你要知道什么时候问题已经足够简单，什么时候优化只是复核而不是救命。

4. 为什么工程里常把 ICP 和 PnP 混着用

原书最后一段非常有工程味：现实里的 RGB-D 并不完美，有些特征有深度，有些没有；有些深度靠谱，有些深度噪声大。所以最实用的思路并不是死守一种误差，而是：

深度已知的特征 → 用 3D-3D 误差
深度未知的特征 → 用 3D-2D 重投影误差

这样就能把 ICP 和 PnP 放到同一个优化框架里。这件事和第六讲的总框架完全一致：只要能写出误差项和雅可比，就能把各种观测拼进同一个最小二乘问题。

这也是第九讲局部地图版 VO 越往后越像“混合前端”的原因。系统不会执着于某一种纯粹的理论形式，而是尽量把手里能用的信息都压榨干净。🌿

课后练习

  1. 为什么 ICP 目标函数在引入质心后，可以把旋转和平移分开求？
    展开查看答案
    因为去质心后，误差函数可拆成“只依赖去质心点相对结构的旋转项”和“只依赖两个质心关系的平移项”。所以先解旋转，再回代平移即可。
  2. 为什么这里强调“匹配关系已知”的 ICP 和很多人印象里的 ICP 不太一样？
    展开查看答案
    因为完整版 ICP 往往还要不断寻找最近邻匹配；而本书这里的视觉场景中，特征匹配已经给出了点对关系，所以只剩下刚体位姿估计本身，问题简单得多。
  3. 第七讲里的 ICP 和第五讲的 RGB-D 点云实践有什么自然联系？
    展开查看答案
    第五讲已经把像素和深度反投影成点云；这一节进一步讨论，当两帧都能产生三维点时，如何直接通过 3D-3D 对齐恢复相机运动。
  4. 为什么工程里会考虑把 PnP 和 ICP 的误差一起放进同一个优化问题？
    展开查看答案
    因为真实数据里并不是每个特征都有高质量深度。把 3D-3D 和 3D-2D 误差混合起来，可以更充分地利用可得信息，提高整体鲁棒性。
