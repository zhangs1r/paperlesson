🎯 本课目标：深入理解 E2Map 的数学定义、情绪建模方法（多元高斯分布）、以及E2Map 的更新机制（事件描述器 → 情绪评估器 → 参数更新）。

1. E2Map 的数学定义

E2Map 本质上是一个三维张量（空间网格地图），每个网格单元存储两类信息：

\(\mathcal{M} \in \mathcal{R}^{\bar{H} \times \bar{W} \times (C_{lang} + C_{emo})}\)
维度含义说明
\(\bar{H}, \bar{W}\)俯视网格地图的尺寸每个网格代表实际空间中的 \(s\) 米
\(C_{lang}\)视觉-语言特征的维度由 LSeg 编码器生成，描述"这个格子里有什么物体"
\(C_{emo}\)情绪参数的维度包含协方差矩阵 \(\Sigma\) 和权重参数 \(w\)
💡 理解关键：E2Map = VLMap（视觉语言地图） + 情绪通道。VLMap 告诉你"哪里有什么"，情绪告诉你"那里感觉怎么样"。

1.1 构建语言特征通道 \(\mathcal{M}_{lang}\)

与 VLMaps [9] 类似，E2Map 首先使用 LSeg [28] 作为预训练的视觉-语言模型，将 RGB 图像编码为视觉-语言特征，然后利用深度图像和相机位姿将每个像素投影到俯视网格地图的对应位置。

1.2 构建情绪通道：多元高斯分布建模

这是 E2Map 最核心的数学创新。论文将情绪建模为高斯分布的加权求和：

\(E(\mathbf{x}) = \sum_{\mathbf{p}_i \in \mathcal{O}} w_{\mathbf{p}_i} \mathcal{N}_{\mathbf{p}_i}(\mathbf{x})\)

其中：

  • \(\mathcal{O}\) = 被占据网格单元的集合（有物体的格子）
  • \(\mathcal{N}_{\mathbf{p}_i}(\mathbf{x})\) = 以网格单元 \(\mathbf{p}_i = [x, y]^\intercal\) 为中心的多元高斯分布
  • \(w_{\mathbf{p}_i}\) = 权重参数
🎯 直觉理解：每个有物体的网格单元都在空间中"散发"一个高斯"情绪场"。场在物体位置最强（中心），越远越弱。多个物体的情绪场叠加起来，就是整张 E2Map 的情绪分布。

每个高斯分布的协方差矩阵初始化为单位矩阵乘以一个系数：

\(\bm{\Sigma}_{\mathbf{p}_i} = \begin{bmatrix} \sigma_x^2 & 0 \\ 0 & \sigma_y^2 \end{bmatrix}\)

权重参数 \(w_{\mathbf{p}_i}\) 初始化为有效高斯分布数量的倒数：

\(w_{\mathbf{p}_i} = \frac{1}{N^{val}_{\mathbf{p}_i}}, \forall \mathbf{p}_i \in \mathcal{O}\)

这样做的目的是避免网格单元的情绪值被高斯分布的数量不成比例地影响——保证公平。

2. 情绪更新机制

当机器人在导航中遇到事件时，E2Map 通过三个步骤更新：

步骤 1
事件描述器
(GPT-4o LMM)
图像序列 → 文本描述
→
步骤 2
情绪评估器
(Llama3 LLM)
文本描述 → 情绪分数
→
步骤 3
E2Map 更新
情绪分数 → 参数更新

2.1 步骤 1：事件描述器（Event Descriptor）

输入三张图像：

  • \(I_{t_{evt}-h}\)：事件发生前
  • \(I_{t_{evt}}\)：事件发生时
  • \(I_{t_{evt}+h}\)：事件发生后
\(l_{evt} = \mathcal{F}_{ed}(I_{t_{evt}-h}, I_{t_{evt}}, I_{t_{evt}+h}, p_{ed})\)

论文让 LMM 采用逐步推理（step-by-step reasoning）：先分别描述每个场景，再整合为对整个事件的完整解释。这种方法在零样本提示中效果很好 [31]。

2.2 步骤 2：情绪评估器（Emotion Evaluator）

输入事件描述 \(l_{evt}\)，输出两个维度的分数：

维度定义分数范围
Upsetness（受挫程度）智能体在给定情境中感受到的影响0–3
Guiltiness（内疚程度）智能体行为对环境造成的影响0–3
\(s_{emo} = s_{upsetness} + s_{guiltiness}\)

论文使用少样本提示（few-shot prompting），给 LLM 提供示例，要求它输出评分及原因。例如：

事件描述："机器人撞到了从墙后走出的人，导致人摔倒。"
评估：Upsetness = 3（机器人受到严重冲击）；Guiltiness = 3（机器人撞倒了人）
情绪分数：6

2.3 步骤 3：E2Map 参数更新

这是更新机制中最精妙的部分。系统需要回答两个问题：更新哪些格子？怎么更新？

Q1: 更新哪些格子？

根据事件发生时智能体的位姿（pose），确定与事件相关的物体/空间所在的 \(n\) 个网格单元位置 \(\mathbf{p}_{update}\)，并计算智能体的朝向单位向量 \(\Vec{\mathbf{v}} = (v_x, v_y)\)。

Q2: 怎么更新？—— 韦伯-费希纳定律

更新规则受 Weber-Fechner 定律 [35] 启发：感觉强度随刺激的对数增长。即：情绪越强，影响范围越大，但增长的幅度逐渐递减。

\(\sigma_k^{new} = \sigma_k \cdot v_k \cdot \log\left(\frac{s_{emo}}{T}\right), \text{ where } k \in \{x, y\}\)

其中 \(T\) 是温度参数（调节敏感度）。关键理解：

  • \(\sigma_k\) 控制高斯分布在 \(x\) 或 \(y\) 方向上的扩散范围
  • 乘以 \(v_k\) 使得朝向方向上的扩散更大（机器人面向的方向影响更远）
  • 取 \(\log(s_{emo}/T)\) 使得情绪越强，扩散越远，但呈对数增长

Q3: 如何保持情绪强度？—— 权重补偿

这里有一个微妙的问题：高斯分布是概率密度函数，在所有区域积分为 1。增大协方差（\(\sigma\)）会让分布在中心点的值变小——这会导致该网格单元的情绪值降低，这显然不是我们想要的。

解决方案：同时更新权重参数来补偿：

\(w_{\mathbf{p}_i}^{new} = w_{\mathbf{p}_i} \cdot \frac{\mathcal{N}_{\mathbf{p}_i}(\mathbf{p}_i)}{\mathcal{N}_{\mathbf{p}_i}^{new}(\mathbf{p}_i)}\)

这样，虽然情绪的影响范围扩大了（通过更新 \(\sigma\)），但该网格单元的情绪强度保持不变（通过更新 \(w\)）。

🎯 整体效果：机器人在某处撞到人 → 该位置的情绪值很高 → 高斯分布的协方差扩大 → 周围一大片区域都受到"负面情绪"影响 → D* 规划器会避开这片区域 → 下次经过时保持安全距离。
