本课目标

把书里的 sphere.g2o 例子拆成“数据文件长什么样”“程序怎么读图”“优化器怎么配”“结果怎么看”四步，不再只把它当成一段会跑的代码。

和前文怎么接

第六讲你已经认识了 g2o，第十讲又用它做了 BA；这一课的难点不在 g2o API 本身，而在于“位姿-位姿约束图”和“李代数节点边”的结合。

书里故意用一个“看起来不真实”的球形轨迹做演示，不是为了炫技，而是为了让你肉眼可判别优化结果。真实轨迹往往很难判断有没有彻底拉直、拉稳；球一旦变形，视觉上特别明显。

这个数据集本质上包含两类边：

  • odometry 边：相邻位姿之间的局部运动约束
  • loop closure 边：跨时间的回环约束
$$ \text{带噪初值} + \text{里程计边} + \text{回环边} \xrightarrow{\text{g2o}} \text{全局一致轨迹} $$

1. 先看原生 g2o 版本：它胜在少做事

原生版本直接使用 g2o 自带的 VertexSE3 和 EdgeSE3。这说明一个重要事实：如果数据格式和内部表示高度对齐，工程实现就会非常省力。

typedef g2o::BlockSolver<g2o::BlockSolverTraits<6,6>> Block;
Block::LinearSolverType* linearSolver =
    new g2o::LinearSolverCholmod<Block::PoseMatrixType>();
Block* solver_ptr = new Block(linearSolver);
auto* solver = new g2o::OptimizationAlgorithmLevenberg(solver_ptr);
g2o::SparseOptimizer optimizer;
optimizer.setAlgorithm(solver);

这里你其实已经能看出第六讲的影子：还是块求解器、还是 LM、还是稀疏优化器。第十一讲真正新增的是“每条边都代表两个位姿之间的相对观测”。

VERTEX_SE3:QUAT 0 tx ty tz qx qy qz qw
EDGE_SE3:QUAT   i j dtx dty dtz dqx dqy dqz dqw info...

这段文件格式非常值得记住。它告诉你：位姿图不是抽象概念，而是一种很朴素的文本图结构。节点存位姿，边存观测和信息矩阵，优化器读进去就能工作。

2. 再看李代数版本：它胜在“你终于能控制误差长什么样”

书里第二个程序更重要，因为它不是为了“再跑一遍球”，而是为了把前一节的误差推导真正落地。这里我们自己定义：

  • 顶点：BaseVertex<6, SE3>
  • 边：BaseBinaryEdge<6, SE3, VertexSE3LieAlgebra, VertexSE3LieAlgebra>
  • 更新方式：左乘更新 _estimate = up * _estimate
virtual void oplusImpl(const double* update) {
  Sophus::SE3 up(
    Sophus::SO3(update[3], update[4], update[5]),
    Eigen::Vector3d(update[0], update[1], update[2])
  );
  _estimate = up * _estimate;
}

这里的好处在于，你不再被库的默认误差定义牵着走，而是可以显式写出：

$$ e = \log\!\left(T_{ij}^{-1} T_i^{-1} T_j\right)^\vee $$
virtual void computeError() {
  Sophus::SE3 v1 = vertex_i->estimate();
  Sophus::SE3 v2 = vertex_j->estimate();
  _error = (_measurement.inverse() * v1.inverse() * v2).log();
}

这一步其实就在告诉我们：位姿图优化并不是什么神秘黑箱，它就是你自己定义误差、自己定义更新，再交给通用最小二乘框架求解。

3. 结果要怎么看：不要只盯着“球变圆了”

“球变圆”当然是最直观的成功信号，但更值得读的是误差下降曲线。书里原生 g2o 版本的 chi2 从十亿量级一路下降到四万量级，说明优化器确实在不断把不一致的位姿约束拉回去。

原生 SE3 版本

优点是实现短、易复现；缺点是你默认接受了库对位姿表示和误差的内部处理方式。

李代数版本

优点是推导和实现一一对应，适合研究和扩展；代价是代码更长，数值细节更容易写错。

书里还提了一个很妙的比较：用李代数自定义实现后，再让 g2o 自带的 SE3 顶点继续优化，最终的度量误差还能再略降一些。这说明“看起来一样圆”不代表“数值已经最优到同一尺度”。

互动判断

如果你准备写论文、想清楚说明自己定义的误差和扰动模型，哪种版本更适合作为起点？

OCR 校正提醒

原始 markdown 中有几处明显识别噪声，例如 result lle.g2o、S03 等写法。这里都应分别理解为 result_lie.g2o、SO3。

4. 本课收束

把这节课学完，你至少该能做到两件事：第一，看到一个 .g2o 文件时，不再只把它当黑盒数据；第二，知道“Pose Graph 优化代码”的本质仍是第六讲那套最小二乘框架，只是变量和误差换了皮囊。

课后练习

  1. 为什么第一个顶点通常要固定住？如果不固定会怎样？
    展开查看答案
    因为整个位姿图只有相对约束，没有绝对参考时会存在整体平移和整体旋转的自由度。固定一个顶点，相当于给整张图选定世界坐标系原点。
  2. 书里同时展示“原生 SE3 顶点”和“李代数顶点”的意义是什么？
    展开查看答案
    前者告诉你快速复现的路径，后者告诉你理论推导如何真正进入代码。两者一起看，才能把“会跑”和“会解释”都补齐。
  3. 如果把 $J_r^{-1}$ 直接近似成单位阵，可能带来什么影响？
    展开查看答案
    在误差较小时往往还能工作，但当残差较大或几何关系更复杂时，收敛速度和精度可能变差，因为你忽略了李群几何对局部线性化的修正。
