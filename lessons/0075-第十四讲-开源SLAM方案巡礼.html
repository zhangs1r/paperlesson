本课目标

不再把这些开源 SLAM 名字当作散乱的专有名词，而是能说出：它们分别诞生在什么背景下、补了上一代系统什么短板。

和前文怎么接

前十三讲我们学的是模块；这一课开始学整机。你会看到不同系统如何把前端、后端、回环和建图拼成自己的架构。

系统主要传感器代表意义
MonoSLAM单目第一个实时单目视觉 SLAM 里程碑
PTAM单目首次明确把 Tracking / Mapping 并行化
ORB-SLAM单目 / 双目 / RGB-D特征点路线的成熟代表
LSD-SLAM单目直接法半稠密 SLAM 的重要里程碑
SVO单目极致轻量的稀疏直接 VO
RTAB-MAP双目 / RGB-D集成度很高的完整应用型系统

1. 特征点路线：从 MonoSLAM 到 ORB-SLAM

MonoSLAM 的意义在于“第一次把实时单目 SLAM 做出来”，而不是“今天还最好用”。它基于 EKF，把相机状态和所有路标一起放进滤波器，是早期滤波派的代表。

PTAM 做了两件影响深远的事：

  • 第一次清晰地区分并并行化了 Tracking 和 Mapping 两条线程
  • 把非线性优化后端真正带进了视觉 SLAM 主流

ORB-SLAM 则是特征点路线的高峰之一。它把我们前面学过的很多模块都做成了工程上很完整的版本：ORB 特征、回环字典、局部 BA、全局 Pose Graph、重定位和三线程协同。

如果你以后想真正精读一套“比较全面、资料多、社区大”的经典代码，ORB-SLAM 仍然是很好的起点。

2. 直接法路线：LSD-SLAM 和 SVO 在追求什么

LSD-SLAM 的贡献，是让“单目直接法 + 半稠密地图 + 回环 + 后端优化”真正组成一个可运行 SLAM。它非常漂亮地连接了第八讲和第十三讲的思想。

SVO 则像另一种人格：它更像一台高帧率 VO 发动机。它把很多东西都做轻了，因此非常快，也因此不是一个完整意义上的 SLAM。

关键词：半稠密、直接法、单目、Sim(3)、CPU 实时。

适合你在意地图比稀疏点更丰富，但又不想完全依赖 GPU 重建的场景。

关键词：稀疏直接法、极快、轻量、适合无人机 / 嵌入式。

适合你首先追求稳定高速 VO，而不是全套回环和全局地图。

关键词：特征点、三线程、重定位强、系统完整。

如果你的目标是全模块理解，通常更适合作为系统级学习样本。

OCR 校正提醒

原始 OCR 把 SVO 展开写成了 Semi-direct Visual Odoemtry，标准写法应为 Semi-direct Visual Odometry。

3. RGB-D 路线：RTAB-MAP 更像“能直接拿来用的系统”

RTAB-MAP 的亮点不是某一个单独算法，而是“它几乎把 RGB-D SLAM 里你会想要的东西都集成进来了”：视觉里程计、词袋回环、位姿图优化、点云 / 网格地图、ROS 生态接入。

这也解释了书里的评价：它非常适合做应用，但不一定是最适合你做研究入门改代码的第一套系统。因为系统太完整、模块太多，改一个地方经常会牵一片。

想学系统架构

优先看 ORB-SLAM。

想学轻量 VO

优先看 SVO。

想学直接法 SLAM

优先看 LSD-SLAM / DSO 这一脉。

想做可交付应用

RTAB-MAP 往往是更现实的起点。

4. 本课收束

这节课最重要的收获，不是“记住几个系统名”，而是开始形成一种读系统的视角：每个系统其实都在“完整性、轻量性、鲁棒性、可扩展性、硬件门槛”之间做取舍。🧵
