1. 本讲知识地图

特征提取

ORB = Oriented FAST + Rotated BRIEF。它把一整张图压缩成可复现、可描述、可匹配的局部锚点。

特征匹配

汉明距离、距离筛选、误匹配控制，是后续所有几何求解的入口，而不是附属步骤。

2D-2D

对极几何负责单目初始化，用本质矩阵从像素对应关系恢复相对位姿，但有尺度歧义，也怕纯旋转。

三角测量

它把 2D 匹配点撑成第一批稀疏 3D 地图点，为后续 PnP 做准备。

3D-2D

PnP 是初始化后的常规主线；线性或闭式解给初值，BA 再减小重投影误差。

3D-3D

ICP 在深度信息足够时直接做刚体对齐，SVD 提供解析解，优化版本则用于统一到同一误差框架里。

2. 三类位姿估计方法的横向比较

路线 输入信息 适合场景 主要优势 主要限制
2D-2D 对极几何 两帧像素对应 单目初始化 不需要深度即可恢复相对运动 有尺度歧义，纯旋转退化
3D-2D PnP 地图点 + 当前像素观测 初始化后的常规跟踪 稳定、直接、适合连续 VO 依赖已有 3D 点质量
3D-3D ICP 两侧都有三维点 RGB-D / 双目点云对齐 信息最充足，SVD 有解析解 依赖双侧深度质量与匹配质量

方法选择速览

优先走 2D-2D。典型场景是单目初始化，还没有现成地图点时只能靠对极几何和三角化先把第一批结构搭起来。

优先走 PnP。只要局部地图已经存在，3D-2D 就比每次重做 2D-2D 更直接、更稳，也更贴近第九讲的真实主循环。

优先考虑 ICP 或混合误差模型。此时你能直接对齐两组 3D 点，甚至把 3D-2D 与 3D-3D 误差一起喂给同一个优化器。

3. 这一讲和前后章节的关系

回扣第三、四讲

没有坐标变换、李群李代数，你几乎无法把位姿估计写成可靠的更新模型。

回扣第五讲

没有针孔相机模型，重投影误差、本质矩阵、PnP 推导都无从谈起。

回扣第六讲

这一讲所有 BA、误差项、雅可比，都是第六讲“状态估计 → 最小二乘 → 优化”框架的具体实例。

通向第八讲

第八讲会给出另一条前端路线：不用描述子匹配，而是直接用像素亮度估计运动。

通向第九讲

第九讲会把这一讲的 ORB、匹配、PnP、局部地图维护，真正封装成一个能连续运行的 VO 系统。

通向第十讲

这里出现的小规模 BA，会在第十讲被放大成真正的后端核心问题。

4. 如果你只复现本讲三件事，优先复现什么

  1. ORB 提取与匹配 demo：这是后面所有位姿估计的入口。
  2. 2D-2D + 三角化 demo：它能帮助你真正理解单目初始化、尺度歧义和纯旋转失败。
  3. EPnP + g2o BA demo：这是第九讲 VO 前端最接近工程主线的一段。

如果你时间足够，再把 3D-3D 的 SVD ICP 跑一遍，会对三种位姿估计的“信息量差异”形成非常具体的直觉。

5. 习题与参考答案

综合练习

  1. 为什么说第七讲是从“模型课”走向“前端系统课”的分界线？
    展开查看答案
    因为前几讲主要在准备数学与成像基础，第七讲则第一次把这些基础真正串成前端估计流程：提特征、做匹配、恢复位姿、建立第一批地图点。
  2. 第七讲的三条位姿估计路线，本质上是按什么顺序展开的？
    展开查看答案
    按可用信息量从少到多展开：先只有两帧像素匹配的 2D-2D，再到已有地图点的一侧深度 3D-2D，最后到两侧都有三维点的 3D-3D。
  3. 为什么单目系统通常只在一开始强依赖 2D-2D，而后续更偏向 PnP？
    展开查看答案
    因为一旦初始化完成并建立了第一批地图点，后续帧就可以直接利用 3D-2D 约束做位姿估计，既更稳定，也摆脱了单目尺度歧义问题。
  4. 第六讲优化思想在第七讲中出现了哪两个最典型的落点？
    展开查看答案
    一个是 PnP 中最小化重投影误差的 BA；另一个是 ICP 的李代数优化写法。两者都把位姿更新、误差项、雅可比和 LM/GN 框架接了起来。
  5. 第七讲和第八讲、第九讲分别是什么关系？
    展开查看答案
    第八讲是与第七讲并列的另一条前端路线，用直接法替代特征点匹配；第九讲则把第七讲的特征法路线工程化，组织成一个完整 VO 系统。
