⚙️ 第 3 课：方法详解（上）—— 文本与图像编码器

LSeg · Method — 文本编码器 · 图像编码器 · 词-像素相关张量 · 训练损失
⏱ 30 分钟 📄 Section 3: Language-driven Semantic Segmentation
🎯 这节课的目标

深入理解 LSeg 方法的前半部分：文本编码器如何嵌入标签、图像编码器如何生成逐像素嵌入、 词-像素相关张量如何计算相似度、以及对比损失如何训练模型。 这节课将包含你第一次看到的完整数学公式！

1. 整体架构一览

先看论文 Figure 2——LSeg 的完整架构图。这是理解整篇论文最重要的图示。

图 2（论文 Figure 2）：LSeg 整体架构。文本编码器将标签嵌入向量空间（蓝色向量）； 图像编码器提取逐像素嵌入（绿色张量）；通过内积计算词-像素相关张量（橙色张量）； 最后空间正则化模块清理和上采样预测结果。

整个流程可以拆解为以下关键步骤：

  1. 文本编码：$N$ 个标签 $\rightarrow$ $N$ 个文本嵌入向量 $T_1, T_2, \dots, T_N \in \mathbb{R}^C$
  2. 图像编码：输入图像 $H\times W$ $\rightarrow$ 下采样后的逐像素嵌入 $I \in \mathbb{R}^{\tilde{H}\times\tilde{W}\times C}$
  3. 相关计算：像素嵌入与文本嵌入做内积 $\rightarrow$ 相关张量 $\tilde{H}\times\tilde{W}\times N$
  4. 损失计算：对每个像素的 $N$ 维向量做 softmax，与 ground truth 计算交叉熵
  5. 空间正则化（下节课）：后处理模块，清理预测结果并上采样到原始分辨率

2. 文本编码器（Text Encoder）

"The text encoder embeds the set of $N$ potential labels into a continuous vector space $\mathbb{R}^C$, producing $N$ vectors $T_1, \dots, T_n \in \mathbb{R}^C$ as outputs."
📝 文本编码器将 $N$ 个候选标签嵌入到连续的 $C$ 维向量空间中，输出 $N$ 个向量。
— Section 3, Method

2.1 核心设计

  • 使用 CLIP 文本编码器（冻结，不参与训练）
  • 为什么冻结？因为 CLIP 已经在大规模图文对上训练好了，它的嵌入空间已经具备了丰富的语义结构
  • 输出的向量集合对输入标签的顺序不变（permutation-invariant）
  • 允许标签数量 $N$ 自由变化——这是灵活性的关键
💡 为什么顺序不变性重要？
传统分割模型输出的第 $i$ 通道固定对应第 $i$ 个类别。LSeg 中，标签的顺序可以任意打乱，输出的分割结果仍然正确。这为动态标签集提供了基础——你可以在推理时新增、删除、重排标签，而模型不需要调整任何参数。

2.2 输入处理

CLIP 文本编码器接受的输入不是原始标签字符串，而是需要特定的 prompt 模板。

"Multiple network architectures are possible, and we use the pretrained Contrastive Language–Image Pre-training (CLIP) throughout."
📝 论文尝试了多种 CLIP 文本编码器变体（ViT-B/32, ViT-B/16, RN$50\times4$, RN$50\times16$），最终选择 CLIP 系列作为标准配置。
— Section 3, Method

论文虽然没有明确说明 prompt 格式，但参考 CLIP 的做法，通常采用 "a photo of {label}" 的模板。例如：

输入标签："dog", "cat", "car", "building"
实际输入："a photo of dog", "a photo of cat", "a photo of car", "a photo of building"
输出：4 个 $C$ 维向量（$C$ 取决于 CLIP 变体，如 ViT-B/32 的 $C=512$）

3. 图像编码器（Image Encoder）

"Similar to the text encoder, the image encoder produces an embedding vector for every input pixel (after downsampling). We leverage dense prediction transformers (DPT) as the underlying architecture."
📝 图像编码器和文本编码器一样，为每个输入像素（下采样后）生成嵌入向量。论文使用 DPT（Dense Prediction Transformer） 作为基础架构。
— Section 3, Method

3.1 为什么用 DPT？

DPT 是一个基于 Vision Transformer 的密集预测架构，它的特殊之处在于：

  • 使用 ViT（Vision Transformer）作为骨干网络
  • 将 ViT 输出的类 token 与 patch token 结合，生成密集预测
  • 通过重组（Reassemble）操作将序列化的 patch 特征恢复为 2D 特征图

LSeg 用 DPT 替代了传统分割模型中的 FCN/DeepLab 风格的解码器。

3.2 输出张量维度

设输入图像大小为 $H \times W$，$s$ 是用户定义的下采样因子（论文中 $s=2$）：

$$\tilde{H} = \frac{H}{s}, \quad \tilde{W} = \frac{W}{s}$$
下采样后的空间尺寸
$$I \in \mathbb{R}^{\tilde{H} \times \tilde{W} \times C}$$
图像嵌入张量：每个空间位置对应一个 $C$ 维向量

$I_{ij} \in \mathbb{R}^C$ 表示像素 $(i, j)$ 的嵌入向量。

📐 数值示例

输入图像：$384 \times 384$
下采样因子 $s=2$
$\tilde{H}=192, \tilde{W}=192$
嵌入维度 $C=512$（ViT-B/32 CLIP）
输出张量：$192 \times 192 \times 512$

🔍 直观理解

每个 $2\times2$ 的像素块（因为 $s=2$）
被映射成一个 $512$ 维的向量。
这个向量应该编码该区域
的视觉语义信息。

4. 词-像素相关张量（Word-Pixel Correlation Tensor）

这是 LSeg 最具创造性的设计：用内积代替传统的线性分类头。

4.1 核心公式

将图像嵌入 $I_{ij}$ 与每个标签嵌入 $T_k$ 做内积：

$$f_{ijk} = I_{ij} \cdot T_k$$
公式 (1)：像素 $(i,j)$ 与标签 $k$ 的相关性

将所有 $N$ 个标签的相关性组合成一个向量：

$$F_{ij} = (f_{ij1}, f_{ij2}, \dots, f_{ijN})^{\mathsf{T}} \in \mathbb{R}^N$$
像素 $(i,j)$ 对所有 $N$ 个标签的相似度向量

所有像素的所有标签构成一个三维张量：$\tilde{H} \times \tilde{W} \times N$（图中的橙色张量）。

🔬 这就是"灵活标签"的数学基础：
传统分割模型中，最后一层线性层的权重矩阵形状是 $C \times K$，其中 $K$ 是固定的类别数，训练完就不能改。

LSeg 中，文本嵌入 $T_k$ 是由 CLIP 动态生成的，$N$ 可以是任意数量、任意顺序。训练时 $N$ 等于训练集的类别数，推理时 $N$ 由用户指定——模型不需要做任何调整！

4.2 与传统方法的对比

🔴 传统分割

$P = \text{softmax}(W \cdot I_{ij} + b)$
其中 $W \in \mathbb{R}^{C \times K}$ 是固定权重
$K$ 是训练时的类别数，无法改变

输出通道固定编号，每个通道
代表一个特定类别

🟢 LSeg

$f_{ijk} = I_{ij} \cdot T_k$
$T_k$ 由 CLIP 文本编码器动态生成
$N$ 可以在推理时任意改变

没有固定权重，标签的语义
由文本嵌入的语义决定

5. 训练损失函数

LSeg 使用 逐像素 softmax + 交叉熵损失，与传统分割一致——但作用的对象是 $F_{ij}$（与文本嵌入的相似度向量），而不是传统分割中的分类 logits。

5.1 逐像素 softmax 目标

给定温度参数 $t$，LSeg 在每个像素位置计算 softmax：

$$\sum_{i,j=1}^{H,W} \text{softmax}_{y_{ij}}\left(\frac{F_{ij}}{t}\right)$$
公式 (2)：逐像素 softmax 目标

其中 $y_{ij}$ 是像素 $(i,j)$ 的真实标签（ground-truth label index），$t$ 是温度参数。这个公式的意思是：对所有像素，最大化其真实类别的 softmax 概率。

5.2 温度参数 $t$

"where $t$ is a user-defined temperature parameter that we set to $t=0.07$."
📝 温度参数设置为 $t=0.07$，这个数值来源于 CLIP 论文和无监督对比学习的常见设置。
— Section 3, Method

温度参数 $t$ 的作用：

  • $t$ 越小，softmax 分布越"尖锐"（更倾向于最高分）
  • $t$ 越大，softmax 分布越"平滑"（各标签概率接近）
  • $t=0.07$ 是比较小的值，鼓励模型更置信地预测

5.3 实际实现

"During training, we minimize a per-pixel softmax with cross-entropy loss (with temperature scaling) as is standard in semantic segmentation. In practice we implement this using the standard nn.CrossEntropyLoss from PyTorch."
📝 论文强调：尽管概念上用了对比学习，但在 PyTorch 实现中直接用标准的 nn.CrossEntropyLoss，因为 CE loss 内部已经包含了 softmax 操作。
— Section 3, Method

PyTorch 的 nn.CrossEntropyLoss 接受形状为 $(N, C, H, W)$ 的 logits（这里是 $F_{ij}$）和形状为 $(N, H, W)$ 的目标标签，内部自动做 softmax 和交叉熵计算。

标签数量 $N$： 4

输入文本 → CLIP 文本编码器 → 4 个 $C$ 维向量

📤 输出张量维度：$\tilde{H} \times \tilde{W} \times 4$

📝 课后练习

📌 练习 1：LSeg 的词-像素相关张量的计算公式是什么？它的形状是多大？

答案：

计算公式：$f_{ijk} = I_{ij} \cdot T_k$，即像素嵌入与文本嵌入的内积。

相关张量的形状是 $\tilde{H} \times \tilde{W} \times N$，其中：

  • $\tilde{H} = H / s$，下采样后的图像高度
  • $\tilde{W} = W / s$，下采样后的图像宽度
  • $N$ = 标签数量（可以自由变化！）

每个像素位置 $(i,j)$ 有一个 $N$ 维向量 $F_{ij}$，表示该像素与所有标签的匹配程度。

📌 练习 2：LSeg 的损失函数与传统语义分割的损失函数有什么异同？

答案：

相同点：两者本质都是逐像素 softmax + 交叉熵损失，在 PyTorch 中都是用 nn.CrossEntropyLoss 实现。

不同点：

  • 传统分割：softmax 的输入是线性分类层的 logits，形状固定为 $C \times K$（$K$ 固定）
  • LSeg：softmax 的输入是 $F_{ij}$（像素与所有标签的内积向量），长度 $N$ 可以变化

此外 LSeg 还有温度缩放（$t=0.07$），这是从对比学习中继承的设计。

📌 练习 3：为什么论文选择冻结文本编码器而不是和图像编码器一起训练？

答案：论文在训练细节中说明：

  • CLIP 文本编码器已经在大规模图文对上预训练过，其嵌入空间已经具备良好的语义结构
  • 冻结文本编码器可以保持这种语义结构的稳定性
  • 如果一起训练，可能会破坏 CLIP 嵌入空间中的语义关系（如"dog"靠近"pet"），降低零样本能力
  • 训练时只更新图像编码器的权重，使其适应 CLIP 的嵌入空间

此外，文本编码器通常参数量较小，冻结它还能节省计算资源。

📌 练习 4（计算题）：假设输入图像为 $512 \times 512$，下采样因子 $s=2$，CLIP ViT-B/32 的嵌入维度 $C=512$，标签数量 $N=20$。请计算：

计算过程：

  1. 下采样后的尺寸：$\tilde{H} = 512/2 = 256$，$\tilde{W} = 512/2 = 256$
  2. 图像嵌入张量 $I$ 的形状：$256 \times 256 \times 512$
  3. 文本嵌入矩阵 $T$ 的形状：$20 \times 512$
  4. 相关张量的形状：$256 \times 256 \times 20$
  5. 内积计算的总操作数：每个像素与所有标签的内积 = $256 \times 256 \times 20 \times 512 \approx 671$ 百万次乘法

相比传统分割（最后一层只有 $512 \times 20 = 10,240$ 个参数），LSeg 的相关计算量更大（每次推理都要动态计算），但获得了灵活性的巨大提升。

📌 练习 5（思考题）：为什么论文说使用 nn.CrossEntropyLoss 就能实现对比学习的目标？

参考答案：

仔细看公式 (1) 和公式 (2)：

$f_{ijk} = I_{ij} \cdot T_k$ 是像素嵌入与文本嵌入的相似度（点积越大越相似）。

对 $F_{ij} = (f_{ij1}, ..., f_{ijN})$ 做 softmax + CE loss，本质就是：

  • 拉近：让 $I_{ij}$ 与正确标签 $T_{y_{ij}}$ 的相似度尽可能大
  • 推远：让 $I_{ij}$ 与其他标签 $T_{k \neq y_{ij}}$ 的相似度相对较小

这正是对比学习的核心思想——只不过对比学习的实现通常需要负样本采样和专门的对比损失函数（如 InfoNCE），而这里 CE loss 自然地充当了对比角色，因为每个像素的"负样本"就是所有其他标签的嵌入。
