🎯 这节课的目标

继续深入 LSeg 的方法：理解 空间正则化模块（BottleneckBlock / DepthwiseBlock）的设计动机和结构， 掌握 标签等变性 这一关键约束，以及 LSeg 的训练配置（初始化、优化器、学习率调度、多数据集混合）。

1. 为什么需要空间正则化？

回顾一下 LSeg 当前的困境：

问题：图像编码器输出的嵌入张量 $I$ 的空间分辨率是 $\tilde{H} \times \tilde{W}$，比原始输入图像 $H \times W$ 低了 s 倍（$s=2$）。 而且由于每个像素的预测是独立的，没有考虑空间连续性，结果可能有噪声。

因此需要两个后处理操作：

  1. 空间正则化：滤除噪声，使预测结果在空间上更平滑
  2. 上采样：将低分辨率预测恢复到原始输入分辨率

但这里有一个关键约束——操作必须对标签顺序保持等变（equivariant）。

2. 标签等变性（Label Equivariance）

"During this process, we have to ensure that all operations stay equivariant with respect to the labels. In other words, there should be no interactions between the input channels, whose order is defined by the order of the words and can thus be arbitrary."
📝 所有操作必须对标签保持等变：输入通道的顺序由标签顺序定义，是任意的，因此通道之间不应有相互作用。
— Section 3, Method

2.1 为什么标准卷积不行？

标准的 2D 卷积 (conv2d) 会在输入通道之间做跨通道混合。对于形状为 $\tilde{H}\times\tilde{W}\times N$ 的相关张量（$N$ 是标签数）：

  • 标准卷积的每个输出通道是所有输入通道的加权和
  • 如果标签顺序从 $[\text{"dog"}, \text{"cat"}]$ 变成 $[\text{"cat"}, \text{"dog"}]$，卷积结果会不同
  • 这就破坏了对标签顺序的等变性

🎯 类比理解：想象你把标签顺序重新排列（就像打乱了一副扑克牌），然后问"哪个标签最匹配这个区域"。 答案应该只取决于标签的内容，而非它们被输入的顺序。

标准卷积就像用手在扑克牌之间来回传递信息——如果你先把牌码好顺序再传递，再打乱顺序，结果会不同。
LSeg 需要的操作是：只在每张牌上独立处理（Depthwise conv），这样无论你以什么顺序发牌，结果都一样。

3. 空间正则化模块

论文设计了两种满足标签等变性的模块：

图 3（论文 Figure 3）：两种空间正则化模块。左：DepthwiseBlock，只有深度可分离卷积 + 非线性激活； 右：BottleneckBlock，额外增加了跨标签的 max-pooling 操作。

3.1 DepthwiseBlock（深度可分离卷积块）

"a simple cascade of depthwise convolutions followed by non-linear activations (DepthwiseBlock)"
📝 DepthwiseBlock 由多个深度可分离卷积级联组成，每个卷积后跟非线性激活（ReLU）。
— Section 3, Method

深度可分离卷积（Depthwise Convolution）与标准卷积的关键区别：

  • 标准 conv2d：每个输出通道 = 所有输入通道的加权组合 → 跨通道交互
  • Depthwise conv2d：每个输出通道 = 仅对应输入通道的空间滤波 → 通道独立

🔴 标准卷积

输入: $C_{in}$ 通道
卷积核: $K_h \times K_w \times C_{in} \times C_{out}$
每个输出通道融合所有输入通道
❌ 对标签顺序不等变

🟢 Depthwise 卷积

输入: $C_{in}$ 通道
卷积核: $K_h \times K_w \times C_{in}$（无 $C_{out}$ 维度）
每个通道独立做空间滤波
✅ 对标签顺序等变 ✅

3.2 BottleneckBlock（瓶颈块）

"another block that additionally augments the depthwise convolutions with the result of a max-pooling operation over the set of labels (BottleneckBlock)"
📝 BottleneckBlock 在深度可分离卷积的基础上，额外增加了跨标签的 max-pooling 操作。
— Section 3, Method

BottleneckBlock 的设计包含两个分支：

  1. 主分支：Depthwise 卷积 + ReLU（保持等变性）
  2. 辅助分支：跨标签维度的 max-pooling（在 $N$ 个标签上取最大值）
  3. 两个分支的结果合并（加和或拼接）

3.3 为什么需要 BottleneckBlock？

💡 设计动机：DepthwiseBlock 虽然满足等变性，但每个通道完全独立，无法利用跨标签的上下文信息。 BottleneckBlock 通过 max-pooling 引入了一个受控的跨标签交互——只传递"哪个标签在当前区域最匹配"这种信息， 而不涉及特定标签对之间的交叉。这样既保留了等变性，又获得了更多信息。

3.4 上采样

"In a final step we use bilinear interpolation to recover predictions at the original resolution."
📝 最后一步使用双线性插值将预测恢复到原始分辨率。双线性插值是逐通道操作的，因此同样满足等变性要求。
— Section 3, Method

5. 训练细节

5.1 初始化

"We initialize the backbone of the image encoder with the official ImageNet pretrained weights from ViT or ResNet and initialize the decoder of DPT randomly."
📝 图像编码器的骨干网络使用 ImageNet 预训练权重（ViT 或 ResNet），DPT 的解码器随机初始化。
— Section 3, Method
🔍 关于 CLIP 图像编码器的说明：论文在脚注中提到，他们也尝试过用 CLIP 图像编码器初始化，但效果不如 ViT 初始化。 这可能是因为 CLIP 图像编码器是为图像级任务优化的，其特征空间不太适合密集预测任务。

5.3 多数据集混合训练

"Our model can be trained on any semantic segmentation dataset and supports flexible mixing of multiple datasets through the text encoder."
📝 LSeg 的一个独特优势：可以同时在多个分割数据集上训练，通过文本编码器来区分不同数据集的标签。
— Section 3, Method

论文在定性实验部分将模型在 7 个数据集联合训练：

🌍 联合训练的数据集

• ADE20K (150 类)
• BDD100K (19 类)
• Cityscapes (19 类)
• COCO-Panoptic (133 类)
• IDD (26 类)
• Mapillary Vistas (66 类)
• SUN RGB-D (37 类)

💡 为什么可以联合训练？

传统分割模型无法直接混合多个数据集，因为不同数据集的标签空间不同（例如 ADE20K 的 "building" 和 Cityscapes 的 "building" 在输出通道中位于不同位置）。

LSeg 通过文本嵌入统一了标签表示——"building" 在 CLIP 空间中总是同一个向量，无论来自哪个数据集！

6. 完整流程回顾

现在我们可以把 LSeg 的完整流程串起来了：

步骤操作输入 → 输出
1标签编码"dog", "cat", ... → $N$ 个 $C$ 维向量 $T_k$
2图像编码$H\times W\times 3$ → $\tilde{H}\times\tilde{W}\times C$ 逐像素嵌入 $I$
3相关计算$f_{ijk} = I_{ij} \cdot T_k$ → $\tilde{H}\times\tilde{W}\times N$ 张量
4空间正则化DepthwiseBlock / BottleneckBlock 处理
5上采样双线性插值恢复到 $H\times W\times N$
6预测$\arg\max$ 每个像素的 $N$ 维向量

📝 课后练习

📌 练习 1：为什么标准卷积不能用在 LSeg 的空间正则化中？

答案：标准卷积会混合不同的输入通道。

在 LSeg 中，相关张量 $\tilde{H}\times\tilde{W}\times N$ 的 $N$ 个通道对应 $N$ 个标签，通道的顺序由标签输入顺序决定。 标准卷积的每个输出通道是所有输入通道的加权组合——如果标签顺序改变，卷积结果也会改变，这破坏了标签等变性。

Depthwise 卷积只在空间维度上滤波，每个通道独立处理，因此对通道顺序保持不变。

📌 练习 2：BottleneckBlock 相比 DepthwiseBlock 多了什么？为什么这能提升性能？

答案：BottleneckBlock 在 DepthwiseBlock 的基础上增加了跨标签的 max-pooling 操作。

这带来一个好处：Max-pooling 可以告诉每个 Depthwise 卷积的通道："在当前空间位置，所有标签中最强的响应是多少"。

这种信息有助于：

  • 抑制模糊区域：如果某个位置对所有标签的响应都很弱，模型可以知道这里可能是未知物体
  • 增强空间一致性：相邻像素的最强标签通常应该一致
  • 保持等变性：max-pooling 对标签顺序不变（无论你以什么顺序排列标签，最大值是一样的）
📌 练习 3：LSeg 如何实现多个数据集的联合训练，而传统方法做不到？

答案：关键在于标签表示的差异。

传统方法：每个数据集的标签空间是独立定义的。例如，Cityscapes 的类别 #{14} = "building"，ADE20K 的类别 #{5} = "building, house"。这两个 "building" 在传统模型的输出层位于不同的通道索引上，无法直接共享。

LSeg：标签是用文本嵌入表示的。无论是来自 Cityscapes 还是 ADE20K，"building" 这个文本经过 CLIP 编码后得到同一个向量。因此模型不需要关心标签来自哪个数据集，只需要学习将图像特征映射到这个统一的文本嵌入空间。

这使得 LSeg 可以轻松地在多个数据集上联合训练，等价于"见过"更多种类的物体，零样本能力更强。

📌 练习 4：论文为什么选择用 ViT/ResNet 的 ImageNet 预训练权重初始化图像编码器，而不是用 CLIP 图像编码器？

答案：论文在脚注中说明：

"We also evaluated on a model initialized with the CLIP image encoder with the same setup and hyperparameters, but observed worse performance than using the ViT initialization."

可能的原因：

  • CLIP 图像编码器是通过对比学习训练的，其最终特征是全局的（对应图像级分类），可能缺乏局部细节信息
  • ViT 的 ImageNet 预训练权重通过分类任务获得，在密集预测任务上通常表现更好
  • DPT 解码器本身就是为了 ViT 特征设计的

这提醒我们：预训练策略的选择需要与下游任务匹配，CLIP 的图像编码器虽然零样本分类能力强，但不一定适合密集预测。

📌 练习 5（思考题）：LSeg 的空间正则化模块的设计理念是什么？还有哪些其他可能的满足等变性的操作？

参考答案：

设计理念是：在不引入标签顺序依赖的前提下，提升预测的空间连续性。

其他可能的等变操作包括：

  • 逐点的 1×1 Depthwise 卷积：最简单的等变操作，只做通道独立的逐点变换
  • 逐通道的空间池化：如 mean-pooling、max-pooling
  • 逐通道的 CRF（条件随机场）：但需要额外的推理步骤
  • 跨标签的 softmax + 逐空间点的独立操作：softmax 是标签数 $N$ 维的，但它对标签排列是等变的（softmax 输出不会随输入通道顺序改变而改变——如果对称地重排输入和输出通道）

论文选择 Depthwise conv 是平衡了表达能力、等变性和计算效率的结果。
