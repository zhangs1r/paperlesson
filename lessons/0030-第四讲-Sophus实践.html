本课目标

不是简单跑通一个样例，而是让你能把前面几节里的数学对象和 Sophus 的 API 一一对上：构造、exp / log、hat / vee、左乘增量更新。

和前后文怎么接

第三讲的 Eigen 几何模块只覆盖了旋转矩阵、四元数等基础对象；第四讲真正需要的是李群 / 李代数操作，所以这里引入 Sophus。后面第十一讲自定义李代数位姿图顶点和边时，还会再次依赖它。

原书这一节的核心很明确：前面所有公式都不该只停留在纸上。比如：

  • 0027 的指数映射，代码里就应该对应一个 exp()
  • 0027 的对数映射，代码里就应该对应一个 log()
  • 0026 的 $\wedge/\vee$ 运算，也应该有直接接口
  • 0029 的左乘扰动更新，程序里也应该真能写成 exp(update) * pose

这正是 Sophus 的价值：它让你不必手写一堆底层矩阵细节，就能把李群和李代数当成一等公民来操作。

1. 安装之前先澄清两个坑：版本和 OCR

原书明确使用的是非模板版本 Sophus，所以给出的获取方式是：

git clone https://github.com/strasdat/Sophus.git
cd Sophus
git checkout a621ff

这一步非常重要，因为后续模板化版本虽然更灵活，但接口和使用细节会有差异。你现在如果只是跟着书学，最好先用书里同一代的版本，把概念吃透再说。

这一节里我顺手修正的几个 OCR 噪声

原始识别文本把 SO3 多处误识别成了 S03 或 S0(3)，还把 1e-4 错打成了 1e-4d。这些如果直接照抄进代码，程序是过不了编译的，所以这里都统一校正成正确写法。

书里的 CMake 配置也很简洁：

find_package(Sophus REQUIRED)
include_directories(${Sophus_INCLUDE_DIRS})
add_executable(useSophus useSophus.cpp)
target_link_libraries(useSophus ${Sophus_LIBRARIES})

它想表达的不是“记住四行模板”，而是让你理解：

  • find_package 负责找到库的头文件和库文件
  • Sophus_INCLUDE_DIRS / Sophus_LIBRARIES 是 cmake 暴露出来的结果
  • 之后你自己的工程也能按这个模式把 Sophus 接进来

2. 示例代码逐段读：每一段都在复现前面课程

分段阅读

#include <Eigen/Core>
#include <Eigen/Geometry>
#include "sophus/so3.h"
#include "sophus/se3.h"

Eigen::Matrix3d R =
    Eigen::AngleAxisd(M_PI / 2, Eigen::Vector3d(0, 0, 1)).toRotationMatrix();
Eigen::Quaterniond q(R);

Sophus::SO3 SO3_R(R);          // 从旋转矩阵构造
Sophus::SO3 SO3_v(0, 0, M_PI / 2); // 从旋转向量构造
Sophus::SO3 SO3_q(q);          // 从四元数构造

这一段对应的其实正是第三讲和第四讲前半段的总汇：同一个几何旋转，可以用旋转矩阵、旋转向量、四元数三种形式来构造，但 Sophus 最终都把它们落成同一个 $SO(3)$ 对象。

Eigen::Vector3d so3 = SO3_R.log();
std::cout << "so3 = " << so3.transpose() << std::endl;

std::cout << "hat =\n" << Sophus::SO3::hat(so3) << std::endl;
std::cout << "vee = "
          << Sophus::SO3::vee(Sophus::SO3::hat(so3)).transpose()
          << std::endl;

这一段就是 0026、0027 两节课的程序版：

  • log() 对应从 $SO(3)$ 回到 $\mathfrak{so}(3)$
  • hat() 对应 $\wedge$，也就是把 3 维向量变成反对称矩阵
  • vee() 则是把反对称矩阵拉回向量

如果你看到这里会觉得“这些 API 很自然”，说明前面两节的数学已经开始内化了。

Eigen::Vector3d update_so3(1e-4, 0, 0);
Sophus::SO3 SO3_updated =
    Sophus::SO3::exp(update_so3) * SO3_R;

这是整节课里最该停下来体会的一句。它不是普通矩阵技巧，而是 0029 左扰动模型的直接代码化：

$$ \boldsymbol{R}_{\text{new}} = \exp(\Delta\boldsymbol{\phi}^\wedge)\boldsymbol{R}_{\text{old}}. $$

也就是说，优化器算出的不是“新的旋转矩阵”，而是一个局部小增量；真正的姿态更新，要靠指数映射把它送回群上，再左乘到旧姿态。

Eigen::Vector3d t(1, 0, 0);
Sophus::SE3 SE3_Rt(R, t);

using Vector6d = Eigen::Matrix<double, 6, 1>;
Vector6d se3 = SE3_Rt.log();

Vector6d update_se3 = Vector6d::Zero();
update_se3(0, 0) = 1e-4;
Sophus::SE3 SE3_updated =
    Sophus::SE3::exp(update_se3) * SE3_Rt;

SE(3) 部分和 SO(3) 是同一套逻辑，只不过现在局部坐标变成了 6 维。书里还特意提醒：Sophus 中 se(3) 的排列是平移在前、旋转在后，这和本书公式保持一致，读代码时就不容易错位。

助教视角的阅读方法

不要把这份示例当成“演示库用法的孤立例子”。更好的读法是：每看到一个 API，就问自己它对应前面哪一个公式。这样以后你再看 g2o、Ceres、GTSAM 或者自己的 VO / BA / 位姿图代码，心里才会有地图。

4. 本课小结

Sophus 的角色

它不是替你“发明新理论”，而是把第四讲已经学过的李群 / 李代数操作变成可直接调用的代码接口。

这节实践最值钱的地方

让你把 exp、log、hat、vee、左乘更新这些数学对象和实际 API 一一对齐。

继续往后看时的抓手

以后再见到 Sophus 相关代码，不要只看“库在怎么写”，而要反向问“它在调用第四讲哪条几何规则”。

课后练习

  1. 为什么书里特意强调使用非模板版本的 Sophus？
    展开查看答案
    因为本书配套代码和讲解以当时的非模板版本为基准，接口更直接、学习曲线更平。模板化版本功能更灵活，但初学阶段会平添不少语法和类型层面的负担，不利于把注意力集中在李群李代数本身。
  2. SO3_R.log() 和 SO3::exp(update) 在数学上分别对应什么？
    展开查看答案
    log() 对应对数映射，把群元素 $SO(3)$ 拉回李代数 $\mathfrak{so}(3)$；exp() 对应指数映射，把局部增量从李代数送回李群。二者合在一起，构成了“局部无约束坐标”和“真实几何姿态”之间的来回翻译。
  3. 为什么说 exp(update) * pose 这类写法会在后面的 BA 和位姿图代码里反复出现？
    展开查看答案
    因为后续优化器解出来的通常都是局部小增量，而不是直接产出新的位姿矩阵。要把这个小增量真正作用到当前估计上，就需要先通过指数映射把它送回群上，再按左乘或右乘约定更新当前位姿。这正是第四讲和后端优化之间最关键的接口。
