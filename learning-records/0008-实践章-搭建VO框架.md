# 第九讲 实践章：搭建 VO 框架

## 日期

2026-06-21

## 核心收获

1. **VO 框架设计**：从零搭建了一个 C++ SLAM 工程，包含 7 个标准目录（bin/include/src/test/lib/config/cmake_modules），以版本号递进（0.1→0.2→0.3→0.4）逐步增加功能。
2. **五个基本类**：Camera（四种坐标变换）、Frame（图像+位姿）、MapPoint（路标点+质量指标）、Map（哈希表管理关键帧和路标）、Config（单例模式读取 YAML 配置）。
3. **两两帧 VO 的局限性**：无结构（Structureless）VO 过度依赖参考帧，容易因误差累积导致漂移和丢失。特征提取+匹配占总耗时 96%。
4. **优化的价值**：在 RANSAC PnP 基础上用 g2o 做一元边优化（仅优化位姿），在几乎不增加计算量的前提下大幅提升估计平稳性。
5. **局部地图管理**：首帧路标初始化 → 三条件剔除（视野/匹配率/视角）→ 动态规模控制（千点阈值）。
6. **工程经验**：参数外部化（YAML）、智能指针内存管理、有限状态机设计、关键帧选取策略。

## OCR 错误纠正

- 原文公式 $\mathbf{T}_{cr}$ 被识别为 $\mathbf { \delta } _ { \mathbf { \mathcal { T } } \nu _ { w } }$，已纠正
- 代码中 `camera2pixel` 函数的参数 `fx_, fy_, cx_, cy_` 被 OCR 误识别为带下标的变量，教材公式原文正确为 $f_x, f_y, c_x, c_y$
